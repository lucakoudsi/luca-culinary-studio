# TELLERDESIGNER – Produktvision

## Zweck
Der Tellerdesigner ist kein Bildgenerator. Er ist ein digitales Food-Styling-Tool für ambitionierte Köche und Fine-Dining-Restaurants. Der Nutzer soll das Gefühl haben, mit einem Creative Director für Anrichten zusammenzuarbeiten. Jede generierte Ansicht soll Inspiration liefern und konkrete Ideen für das Anrichten geben.

## Zielgruppe
Fine Dining, Gourmetrestaurants, Privatköche, Catering, Kochschulen, ambitionierte Hobbyköche. Nicht für Social-Media-Bilder. Nicht für KI-Spielereien. Sondern für echte Tellergestaltung.

## Grundprinzip
Der Fokus liegt immer auf dem Teller. Nicht auf Formularen. Nicht auf Einstellungen. Nicht auf Buttons. Die UI soll den Blick automatisch auf das Gericht lenken.

## Emotion
Der Nutzer soll denken: „Das sieht aus wie eine Seite aus einem Michelin-Magazin." Nicht: „Ich benutze einen KI-Generator."

## Designphilosophie
Die Seite muss wirken wie eine Mischung aus Apple, W² Recipes, Michelin Guide, Aesop, Luxuskochbuch. Eigenschaften: extrem viel Weißraum, ruhige Farben, elegante Typografie, dünne Linien, hochwertige Schatten, keine überladenen Karten, keine bunten Farben, keine Standard-Dashboard-Optik.

## Das Tellerbild ist der Held
Das Tellerbild ist das größte Element der Seite. Alles andere unterstützt dieses Bild. Die Annotationslinien erklären das Design. Nicht dekorativ. Sondern informativ.

## Ablauf
Beim Generieren soll sich die Seite wie ein Designer anfühlen. Nicht wie eine API. Der Nutzer soll das Gefühl haben, dass die KI gerade ein Plating entwickelt. Animationen langsam. Ruhig. Elegant.

## Animationen
Subtil. Keine schnellen Popups. Keine Bounce-Effekte. Keine übertriebenen Bewegungen. Alles soll leicht wirken. Fast wie eine Produktseite von Apple.

## Mikroanimationen
Hover auf Annotation → Komponente auf dem Teller hervorheben. Hover auf Komponente → passende Annotation hervorheben. Hover auf Buttons → sanfte Farbänderung. Keine harten Übergänge.

## Typografie
Headlines: Elegant, leicht editorial, großzügige Abstände. Texte: Zurückhaltend, nicht fett, keine lauten Farben.

## Layout
Die Seite muss luftig wirken. Große freie Flächen. Der Teller bekommt Platz zum Atmen. Keine Elemente dürfen den Teller optisch bedrängen.

## Komponenten
Linke Seite: Rezept, Stil, Schwierigkeit, Fokus, Generieren. Rechte Seite: Hero-Tellerbild, Annotationen, Designinformationen, Varianten.

## Varianten
Die Varianten sollen wie kleine Fotografien wirken. Nicht wie Karten. Beim Anklicken soll der Hauptteller elegant wechseln.

## Qualität
Alles soll wirken, als hätte ein UI-Designer mehrere Wochen daran gearbeitet. Keine Standard-ShadCN-Komponenten ohne Anpassung. Alle Komponenten müssen sich wie aus einem Guss anfühlen.

## Erweiterbarkeit (Roadmap – später ergänzbar)
Verschiedene Tellerformen, dunkler Hintergrundmodus für schwarze Teller, mehrere Kameraperspektiven (Top, 45°, Detail), Vorher/Nachher-Vergleich, Anrichteschritte als Animation, Zutaten-Hotspots mit Beschreibungen, PDF-Export im W²-Stil, Speichern von Designs, Designhistorie, Variantenvergleich nebeneinander.

## Leitsatz für jede Entscheidung
Bei jeder Entscheidung frage dich: „Macht diese Änderung den Teller zum Mittelpunkt der Seite oder lenkt sie davon ab?" Wenn sie ablenkt, ist sie falsch. Wenn sie den Teller stärker in Szene setzt, ist sie richtig.
